## 1. 로그인 / 로그아웃

### 로그인
1. 브라우저를 열고 사장님이 알려주신 주소로 들어가세요.
2. 로그인 화면에서 사장님이 만들어 준 **이메일**과 **비밀번호**를 입력하세요.
3. **로그인** 버튼을 누르면 발송 화면으로 들어갑니다.

### 로그아웃
- 화면 맨 위에 있는 **"로그아웃"** 글씨를 누르세요.
- 모바일(폰)에서는 우측 상단의 **☰** 버튼을 먼저 누르면 로그아웃 버튼이 나옵니다.

> 💡 **중요**: 공용 컴퓨터(예: 매장 카운터)에서 로그인했다면 일 끝나고 꼭 로그아웃하세요.

## 2. 화면 구성

로그인하면 **발송** 화면이 바로 열립니다. 직원 계정에는 메뉴가 "발송" 하나밖에 없어요.

화면 위쪽에는 **탭**이 두 개 있습니다:

- **대기** 탭: 아직 발송 안 한 거래 목록. 여기서 발송 처리를 합니다.
- **완료** 탭: 이미 발송 처리한 거래 목록. 송장번호 수정이나 취소를 할 수 있어요.

탭 옆에 숫자가 표시돼 있어서 몇 건이 대기 중인지 한눈에 볼 수 있어요.

---

## 3. 발송 대기 확인

1. **대기** 탭을 클릭하세요 (기본으로 열려 있어요).
2. 화면에 **거래처별로 묶여서** 대기 중인 거래가 나옵니다.
   - 예: "홍길동마트" 아래에 주문한 상품 3개가 한 그룹으로 표시됨
3. 각 거래는 다음 정보를 보여줍니다:
   - 주문 날짜
   - 상품명
   - 수량
   - 옵션 (색상, 사이즈 등이 있으면)

> ⚠️ **금액이나 단가는 직원 화면에 표시되지 않습니다.** 정상입니다.

---

## 4. 발송 처리 (송장번호 입력)

### 단계별 방법

1. **대기** 탭에서 오늘 발송할 거래들을 **체크박스로 선택**하세요.
   - 같은 거래처의 여러 거래를 한 번에 선택할 수 있어요.
   - 거래처 이름 옆의 체크박스를 누르면 그 거래처의 거래 전체가 선택됩니다.

2. 화면 위쪽(또는 아래쪽)의 입력란을 채우세요:
   - **택배사**: 기본값은 "로젠택배"로 설정돼 있어요. 다른 택배사면 바꿔주세요. (예: CJ대한통운, 한진택배)
   - **송장번호**: 실제 운송장 번호를 입력하세요. (예: `1234567890`)
   - **비고** (선택): 특이사항이 있으면 적어주세요. 없으면 비워두세요.

3. **발송 처리** 버튼을 누르세요.
4. "발송 처리 완료" 같은 메시지가 뜨면 성공입니다.
5. 처리된 거래는 **대기** 탭에서 사라지고 **완료** 탭으로 이동합니다.

### 꿀팁 💡
- 같은 택배사 + 같은 송장번호로 여러 상품을 한꺼번에 보내는 경우 → 그 거래들을 **모두 선택한 후** 한 번에 처리하면 편해요.
- 거래처마다 송장번호가 다르면 → 거래처 하나씩 따로 처리하세요.

## 5. 직접 수령인 경우

손님이 매장에 직접 찾아가는 경우 (택배를 안 쓰는 경우):

1. 대기 탭에서 해당 거래를 선택하세요.
2. **송장번호 칸을 비워둔 채로** 발송 처리 버튼을 누르세요.
3. 자동으로 "직접수령"으로 처리됩니다.

> 💡 송장번호가 없어도 발송 처리는 됩니다. 비우기만 하면 돼요.

---

## 6. 발송 안내 메시지 복사

발송 처리 후 손님에게 카톡으로 알려줄 때 쓰는 기능입니다.

### 거래처별 전체 안내 (한꺼번에)

1. **완료** 탭으로 이동하세요.
2. 해당 거래처 그룹의 **"전체 안내 복사"** 버튼을 누르세요.
3. "복사됨!" 이라고 잠깐 표시되면 복사 완료입니다.
4. 카톡 창을 열고 **붙여넣기** (Ctrl+V 또는 길게 눌러서 붙여넣기) 하세요.
5. 자동으로 아래 같은 메시지가 만들어져 있어요:

   ```
   상품A 2개, 상품B 1개
   발송했습니다. 송장번호: 1234567890
   배송조회: https://www.ilogen.com/web/personal/trace/1234567890
   ```

### 거래 하나씩 안내

- 개별 거래 옆의 **"안내 복사"** 버튼을 누르면 그 거래 하나만 메시지로 복사돼요.

### 주의사항
- 같은 거래처에 **송장번호가 다른 거래가 섞여 있으면** 자동으로 거래별로 나눠서 표시돼요.
- **직접 수령**으로 처리한 건은 "송장번호" 대신 "직접수령"으로 메시지가 만들어집니다.

## 7. 잘못 발송 처리한 경우 (취소)

실수로 엉뚱한 거래를 발송 처리했을 때:

1. **완료** 탭으로 이동하세요.
2. 취소할 거래를 **체크박스로 선택**하세요.
3. 화면 위쪽의 **"선택 취소"** 버튼을 누르세요.
4. "정말 취소하시겠습니까?" 라는 확인 창이 뜨면 **확인**을 누르세요.
5. 취소된 거래는 다시 **대기** 탭으로 돌아갑니다.

> ⚠️ 사장님과 관련된 작업이므로, 가능하면 취소 전에 한 번 사장님께 알려주세요.

---

## 8. 송장번호 수정

송장번호를 잘못 입력한 경우:

1. **완료** 탭으로 이동하세요.
2. 수정할 거래의 **송장번호 칸**을 클릭하세요.
3. 입력창이 나타나면 올바른 번호로 고치세요.
4. **저장** 버튼을 누르세요.
5. **비고** 칸도 같은 방식으로 수정할 수 있어요.

## 9. 자주 묻는 질문

### Q1. 화면에 "권한이 없습니다"라고 떠요.
→ 사장님께 "직원 계정 권한을 확인해 주세요"라고 말씀드리세요. `user_roles` 테이블에 계정 등록이 안 돼 있을 수 있어요.

### Q2. 대기 탭에 아무것도 없어요.
→ 오늘 발송할 주문이 없거나, 사장님이 아직 거래를 입력하지 않은 거예요. 정상입니다.

### Q3. 송장번호를 잘못 입력하고 발송 처리했어요.
→ **7번(취소)** 대신 **8번(송장번호 수정)**으로 고치면 됩니다. 이미 카톡으로 안내까지 보냈다면 수정 후 다시 안내 복사해서 보내주세요.

### Q4. 손님이 상품을 직접 받으러 왔어요. 어떻게 처리해요?
→ **5번(직접 수령)** 방법대로 송장번호를 비워두고 처리하면 됩니다.

### Q5. 관리자 메뉴(거래, 명세서 등)는 왜 안 보여요?
→ 직원 계정은 발송만 볼 수 있도록 설정돼 있어요. 이건 정상이고 의도된 동작입니다. 가격 정보 보호를 위해서예요.

### Q6. 모바일(폰)에서도 쓸 수 있어요?
→ 네, 가능합니다. 다만 표가 많아서 화면이 작으면 좌우로 스크롤해야 할 수 있어요. 가급적 PC나 태블릿 사용을 권장합니다.

### Q7. 로그인이 안 돼요.
→ 비밀번호를 정확히 입력했는지 확인하세요. 계속 안 되면 사장님께 비밀번호 재설정을 요청하세요.

## 업무 순서 요약 (치트시트)

```
[아침]
1. 로그인
2. 대기 탭 확인 → 오늘 발송할 주문 파악

[발송 작업]
3. 상품 포장
4. 택배사에 넘기고 송장번호 받기
5. 대기 탭에서 거래 선택
6. 택배사 + 송장번호 입력
7. 발송 처리 버튼

[마무리]
8. 완료 탭에서 "전체 안내 복사"
9. 손님에게 카톡으로 발송 안내
10. 로그아웃
```

궁금한 점이나 문제가 생기면 언제든 사장님께 문의하세요. 👍
