## 1. 시작하기 전에

### 이 시스템으로 할 수 있는 것
- 거래처(손님) 정보 관리
- 상품과 가격 관리 (등급별/특별단가/옵션별)
- 거래 기록 (손으로 입력하거나 AI가 자동 인식)
- 명세서 자동 생성 + PNG/PDF 다운로드
- 입금 관리 + 미수금 현황
- 발송 관리 + 택배 안내 자동 복사
- 엑셀로 데이터 내보내기
- 직원 계정 분리 (직원은 발송만 볼 수 있음)

### 용어 안내 (쉬운 설명)
- **거래처**: 물건을 사 가는 손님 (예: "홍길동마트")
- **가격 등급**: 거래처별로 다른 도매가를 적용하기 위한 분류 (예: VIP 등급, 일반 등급)
- **특별단가**: 특정 거래처에게만 주는 특별 가격 (등급을 무시하고 적용됨)
- **옵션**: 상품의 색상/사이즈 같은 변형 (예: "빨강 / M")
- **명세서**: 여러 거래를 한 장에 정리한 청구서
- **미수금**: 아직 못 받은 돈

## 2. 로그인

1. 브라우저에서 사이트 주소에 접속하세요.
2. 이메일과 비밀번호를 입력하고 **로그인** 버튼을 누르세요.
3. 로그인 후에는 자동으로 **홈(대시보드)** 화면이 열립니다.

## 3. 홈 (대시보드)

상단 메뉴에서 **홈**을 누르면 나옵니다. 오늘 하루가 어땠는지 한눈에 보는 화면이에요.

### 보이는 정보
- **오늘 매출 / 이번 주 매출 / 이번 달 매출**
- **마진 (총 이익)**: 매출에서 원가를 뺀 금액
- **마진율**: 매출 대비 이익 비율 (%)
- **지난달 대비 변화**: 매출이 얼마나 늘었는지/줄었는지

### 마진 계산 시작일
마진은 원가가 입력된 상품만 계산에 들어갑니다. 아직 원가 입력이 완료 안 된 상태라면:

1. **설정 → 회사 정보**로 이동
2. **마진 계산 시작일**을 오늘 날짜로 설정
3. 그 이후 거래만 마진에 포함됨

> 💡 이 날짜 이전 거래는 마진 계산에서 빠집니다. 원가 입력이 다 끝나면 이 날짜를 더 앞으로 당기면 돼요.

## 4. 거래처 관리

### 거래처 등록
1. 상단 메뉴 **거래처** 클릭
2. **+ 새 거래처** 버튼
3. 입력:
   - **이름** (필수): 거래처 이름
   - **전화번호** (선택)
   - **주소** (선택)
   - **가격 등급**: 기본으로 적용할 등급 (없으면 소비자가 적용)
   - **비고** (선택)
4. **저장**

### 거래처 정보 수정
1. 거래처 목록에서 해당 거래처 클릭
2. 상세 화면에서 정보 수정 후 **저장**

### 특별단가 등록 (특정 상품만 싸게 주고 싶을 때)
1. 거래처 상세 화면 → **특별단가** 섹션
2. **+ 상품 추가** 버튼
3. 상품을 고르고 특별 가격 입력
4. **저장**

> 💡 특별단가는 가격 등급보다 **우선**합니다. 즉 VIP 등급이어도 특별단가가 있으면 특별단가로 팔려요.

### 거래처 삭제 (비활성화)
1. 거래처 상세 화면에서 **삭제** 버튼
2. 삭제 확인 창에서 확인
3. 목록에서는 사라지지만 **데이터는 안 없어져요**. 복구 가능합니다. ([16번 참고](#16-데이터-복구-실수로-삭제한-경우))

## 5. 상품 관리

### 상품 등록
1. 상단 메뉴 **상품** 클릭
2. **+ 새 상품** 버튼
3. 입력:
   - **이름** (필수)
   - **분류** (선택): 예) "큐대", "용품" 등
   - **비고** (선택)
4. **저장**
5. 저장 후 **상세 화면**에서 가격/옵션 설정

### 가격 설정 (등급별)
1. 상품 상세 화면 → **가격** 섹션
2. 각 등급별로 가격 입력:
   - **원가**: 이 값이 있어야 마진 계산이 됩니다.
   - **소비자가**: 등급 없는 손님(또는 기본 거래처)에게 적용되는 가격
   - **등급별 가격**: 각 가격 등급(VIP/일반 등)에 해당하는 가격
3. **저장**

> ⚠️ 가격을 바꾸면 **변경 이력**에 자동으로 기록돼요. 상품 상세 하단의 **가격 변동 이력**에서 볼 수 있어요.

### 옵션 설정 (색상, 사이즈 등)
1. 상품 상세 → **옵션** 버튼
2. 옵션 이름 입력 (예: "색상")
3. 옵션 값 입력 (예: "빨강, 파랑, 검정")
4. **가격에 영향**: 옵션별로 가격이 다르면 체크
5. **필수**: 거래 입력 시 반드시 골라야 하면 체크
6. **저장**

### 옵션별 가격 (가격에 영향이 있는 경우)
1. 상품 상세 → **옵션별 가격** 섹션
2. 각 옵션값 + 등급 조합마다 가격 입력

### 상품 삭제
- 거래 기록이 있는 상품은 삭제되지 않습니다. (데이터 보호)
- 거래 기록이 없는 상품만 완전 삭제됩니다.

## 6. 거래 입력 (수동)

AI를 안 쓰고 직접 입력하고 싶을 때 쓰는 메뉴입니다.

### 새 거래 입력
1. 상단 메뉴 **거래 → + 새 거래**
2. 입력:
   - **날짜**: 주문 받은 날짜
   - **거래처**: 드롭다운에서 선택
   - **상품**: 드롭다운에서 선택
   - **옵션**: 상품에 옵션이 있으면 선택 (필수 옵션은 ⚠️ 표시)
   - **수량**: 개수 입력
   - **단가**: 자동으로 계산돼서 채워짐 ⭐
3. **저장**

### 단가 자동 계산 규칙 (우선순위)
1. **특별단가** (거래처별 설정된 가격)
2. **옵션별 가격** (옵션에 가격 영향 있을 때)
3. **등급 단가** (거래처 가격 등급 기준)
4. **소비자가** (위 전부 없을 때)

→ 단가 옆에 **"특별단가"** / **"등급 단가"** 같은 출처 표시가 나타나요.

### 가격 인상 안내
이전 거래와 비교해서 단가가 올랐다면 자동으로 안내 문구가 나옵니다. "명세서에 표시" 체크박스를 켜면 명세서 비고란에 자동으로 들어가요.

### 거래 수정/삭제
- 거래 상세 화면에서 수정 가능
- **단, 명세서에 포함된 거래는 수정/삭제가 막힙니다.** (DB 트리거로 보호)
- 명세서에서 해당 거래를 빼거나 명세서를 삭제해야 수정 가능합니다.

## 7. 주문 인식 (AI 자동)

카톡/문자/메모 등으로 받은 주문 메시지를 **AI가 자동으로 읽어서** 거래로 변환하는 기능입니다.

### 사용 방법
1. 상단 메뉴 **주문인식**
2. **거래처 선택** 드롭다운에서 주문한 거래처 선택
3. **주문 메시지** 칸에 받은 텍스트를 붙여넣기
   - 예: `"안녕하세요, 큐대 2개랑 초크 5개 보내주세요"`
4. **AI 인식** 버튼 클릭
5. 몇 초 뒤 자동으로 **상품 매칭** 결과가 표시됩니다.

### 인식 결과 확인
- 상품명 옆에 **매칭 상품** 표시
- 매칭이 잘못되었으면 **변경** 버튼으로 다른 상품 선택
- 수량/옵션/단가 수정 가능
- 거래별로 **비고** 입력란 있음

### 가격 자동 적용
- 거래처의 단가가 자동으로 채워짐
- 가격 인상이 있으면 **"명세서에 표시"** 체크박스로 안내 가능

### 저장
1. 모두 확인 후 **일괄 저장** 버튼
2. 전체가 한 번에 저장됨 (하나라도 실패하면 전체 롤백)

### 중복 방지
같은 메시지를 두 번 인식하면 **"이미 처리된 메시지"** 경고가 뜹니다.

### 별칭 자동 저장
AI가 "큐대"를 "8온스 큐대" 상품과 매칭하면, 그 별칭이 자동으로 저장돼서 **다음 번에 더 빨리 인식**됩니다.

## 8. 별칭 관리

손님이 말하는 상품 이름(예: "초크")을 실제 상품(예: "마스터 초크 4g")과 연결해주는 기능입니다.

### 별칭 확인/검색
1. 상단 메뉴 **별칭**
2. 전체 별칭 목록이 나옵니다.
3. 검색창으로 특정 단어 찾기 가능

### 별칭 추가
1. 상품 상세 화면 → **별칭** 섹션
2. 별칭 입력 후 추가
3. 다음 주문부터 AI가 이 별칭을 알아서 매칭

### 별칭 삭제
- 별칭 목록에서 X 버튼으로 삭제

## 9. 발송 관리

### 대기 탭 (발송해야 할 거래)
1. 상단 메뉴 **발송 → 대기** 탭
2. 거래처별로 그룹지어 표시됨
3. 발송할 거래 선택
4. **택배사 / 송장번호 / 비고** 입력
5. **발송 처리** 버튼

### 송장번호 없이 처리 (직접 수령)
- 송장번호 칸 비운 채로 발송 처리 → "직접수령"으로 자동 처리

### 완료 탭
- 발송 처리된 거래 확인
- **송장번호 / 비고** 인라인 수정 가능
- **안내 복사** 버튼 → 손님에게 보낼 카톡 메시지 자동 생성
- **전체 안내 복사** → 거래처의 모든 거래를 하나의 메시지로 묶음
- **선택 취소** → 체크박스 선택 후 일괄 취소 (취소하면 대기 탭으로 복귀)

### 배송 방법에 따른 색상 구분
- **택배** (파란 뱃지): 송장번호가 있음
- **직접수령** (녹색 뱃지): 손님이 직접 가져감

> 💡 직원 계정도 이 페이지를 쓸 수 있어요. 단, 가격 정보는 보이지 않습니다.

## 10. 명세서

### 명세서 만들기
1. 상단 메뉴 **명세서 → + 새 명세서**
2. 거래처 선택
3. **시작일 / 종료일** 입력
   - 시작일 기본값: 이 거래처의 명세서에 포함 안 된 거래 중 가장 오래된 날짜
4. 미리보기에서 포함될 거래 확인
5. **생성** 버튼
6. 명세서 번호가 자동 발급되고 상세 화면으로 이동

### 명세서 상세
- 거래 목록 전체
- **이전 미수금** (기존에 밀린 금액)
- **이번 명세서 합계**
- **총 미수금** (이전 + 이번)
- **배송 상태 뱃지** (배송전/부분배송/배송완료)

### PNG / PDF 다운로드
- **PNG 다운로드** 버튼: 이미지 파일로 저장 (카톡으로 보내기 편함)
- **PDF 다운로드** 버튼: 인쇄용 PDF 저장

### 명세서 삭제
- 삭제하면 포함된 거래들이 **다시 명세서에 포함 가능한 상태**로 돌아옵니다.
- 삭제 후 복구도 가능합니다. ([16번 참고](#16-데이터-복구-실수로-삭제한-경우))

---

## 11. 입금 관리

### 입금 등록
1. 상단 메뉴 **입금 → + 새 입금**
2. 입력:
   - **거래처**
   - **입금 날짜**
   - **금액**
   - **입금 방법** (선택): 예) "계좌이체", "현금"
   - **비고** (선택)
3. **저장**

### 주의사항
- 금액이 0 이하면 저장 안 돼요.

### 입금 수정/삭제
- 목록에서 해당 입금 클릭 → 수정 또는 삭제

---

## 12. 미수금 현황

### 보이는 정보
1. 상단 메뉴 **미수금**
2. 거래처별로 표시:
   - **총 명세서 금액**
   - **총 입금액**
   - **미수금** (= 명세서 - 입금)
3. 미수금이 많은 순으로 정렬

### 필터
- **미수금 있는 곳만 보기**
- **완납된 곳 보기**

> 💡 거래처를 삭제해도 미수금이 남아있으면 여기 표시됩니다. (돈은 받아야 하니까요)

---

## 13. 엑셀 내보내기

### 사용 방법
1. 상단 메뉴 **내보내기**
2. 다운로드할 데이터 선택:
   - **상품 데이터** (등급별 가격, 판매량, 매출액 포함)
   - **거래처 데이터** (누적 거래액, 거래 건수, TOP 상품 포함)
   - **거래 내역** (기간 선택 필요)
3. **다운로드** 버튼 클릭
4. 엑셀 파일(.xlsx)이 다운로드됩니다.

### 활용
- 세무사에게 전달
- 백업 용도
- 엑셀에서 추가 분석

## 15. 직원 계정 만들기

직원에게 **발송 권한만** 주고 싶을 때 쓰는 방법입니다. (가격/거래 등 민감 정보는 안 보임)

### 1단계: 직원 계정 생성
1. **Supabase 대시보드** 접속 (supabase.com → 프로젝트 선택)
2. 좌측 **Authentication → Users** 메뉴
3. **Add user → Create new user** 버튼
4. 직원의 이메일과 임시 비밀번호 입력
5. **Create user**
6. 목록에서 방금 만든 계정의 **UID**(uuid 형태)를 복사

### 2단계: 역할 등록 (SQL)
1. Supabase 대시보드 좌측 **SQL Editor** 메뉴
2. 아래 쿼리에서 UID만 바꿔서 실행:

   ```sql
   insert into user_roles (user_id, role) values
     ('여기에_직원_UID_붙여넣기', 'staff')
   on conflict (user_id) do update set role = excluded.role;
   ```

3. 성공하면 직원이 로그인할 수 있습니다.

### 확인
1. 시크릿 창에서 직원 계정으로 로그인
2. 메뉴에 **발송**만 보이면 성공
3. 다른 URL 직접 입력해도 접근 차단돼야 함

### 직원 내보내기
- SQL Editor에서:
  ```sql
  delete from user_roles where user_id = '직원_UID';
  ```
- 또는 Supabase 대시보드에서 계정 자체 삭제

## 16. 데이터 복구 (실수로 삭제한 경우)

이 시스템은 **Soft Delete** 방식입니다. 삭제해도 DB에는 남아있어요 (`is_active = false`로만 바뀜).

### 복구 방법
1. **Supabase 대시보드** → **Table Editor**
2. 해당 테이블 선택:
   - `customers` (거래처)
   - `products` (상품)
   - `invoices` (명세서)
   - `payments` (입금)
   - `price_tiers` (가격 등급)
3. 삭제된 행 찾기 (`is_active = false`인 것)
4. `is_active`를 **true**로 변경
5. 저장

> 💡 화면에서는 복구 버튼이 없어요. 꼭 Supabase 대시보드에서 직접 해야 합니다. 자주 생기는 일이 아니라 그렇게 설계됐어요.

## 17. 자주 묻는 질문

### Q1. AI 주문 인식이 엉뚱한 상품으로 매칭해요.
→ 인식 결과 화면에서 **"변경"** 버튼으로 올바른 상품 고르면 돼요. 한 번 고쳐주면 **별칭으로 자동 저장**돼서 다음부턴 제대로 인식합니다.

### Q2. 거래를 수정하려는데 수정이 안 돼요.
→ 그 거래가 **명세서에 포함**되어 있으면 수정이 막혀요. 명세서에서 빼거나 명세서 자체를 삭제해야 수정 가능합니다.

### Q3. 대시보드의 마진이 이상해요.
→ **설정 → 마진 계산 시작일**을 확인하세요. 원가 입력이 안 된 상품이 있으면 마진 계산에서 빠집니다.

### Q4. 모바일에서 쓸 수 있어요?
→ 네, 반응형이라 폰에서도 쓸 수 있어요. 다만 거래 입력 같은 복잡한 작업은 PC가 편합니다.

### Q5. 명세서를 손님에게 보낼 때 어떻게 해요?
→ 명세서 상세에서 **PNG 다운로드** → 카톡으로 이미지 보내기가 가장 편해요.

### Q6. 가격을 실수로 바꿨어요. 이전 가격이 뭐였는지 알 수 있나요?
→ 네, **상품 상세 하단의 "가격 변동 이력"** 섹션에서 언제 어떤 가격이었는지 전부 볼 수 있어요.

### Q7. 직원이 가격을 볼 수 있나요?
→ 아니요. 발송 페이지는 원래 가격/금액 표시가 없어요. 직원 계정은 발송 페이지만 접근 가능하므로 가격 정보가 완전히 차단됩니다.

### Q8. 주문인식에서 "이미 처리된 메시지"가 뜨는데 다시 인식하고 싶어요.
→ Supabase 대시보드 → `order_messages` 테이블 → 해당 행 삭제 후 다시 시도.

### Q9. 특별단가를 등록했는데 적용 안 돼요.
→ 가격 우선순위는 **특별단가 > 옵션별 > 등급 > 소비자가** 입니다. 특별단가가 있으면 최우선이에요. 혹시 다른 상품이나 다른 거래처에 등록한 건 아닌지 확인해 주세요.

### Q10. 로그아웃 후에도 다시 자동 로그인되는데요?
→ 브라우저가 세션을 기억해서 그래요. 완전히 끊고 싶으면 브라우저의 쿠키/사이트 데이터 삭제를 쓰세요.

```
[아침]
```

### 데이터 안전 팁
- 중요한 변경 전에는 **엑셀 내보내기**로 백업
- 상품 가격 대량 변경 시에는 **가격 변동 이력**이 남으므로 걱정 없음
- 삭제는 Soft Delete라 복구 가능 ([16번 참고](#16-데이터-복구-실수로-삭제한-경우))

### 성능 팁
- 거래가 많아지면 (연 단위) 목록 페이지가 느려질 수 있어요. 그땐 기간 필터를 좁히세요.
- AI 주문 인식은 긴 메시지일수록 오래 걸려요. 상관없는 부분은 잘라서 붙여넣으면 더 빨라요.
